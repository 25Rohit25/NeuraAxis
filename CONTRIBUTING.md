# Contributing to NEURAXIS

Thank you for your interest in contributing to NEURAXIS! This document provides guidelines and information for contributors.

## Code of Conduct

Please read and follow our Code of Conduct to maintain a welcoming and inclusive environment.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Set up the development environment (see README.md)
4. Create a feature branch

## Development Workflow

### Branch Naming

- `feature/description` - New features
- `fix/description` - Bug fixes
- `docs/description` - Documentation updates
- `refactor/description` - Code refactoring

### Commit Messages

Follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:

```
type(scope): description

[optional body]

[optional footer]
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

### Pull Requests

1. Ensure all tests pass
2. Update documentation as needed
3. Add relevant tests for new features
4. Request review from maintainers

## Code Style

### TypeScript/JavaScript
- Use ESLint and Prettier configurations
- Follow the existing code style
- Write meaningful variable and function names

### Python
- Follow PEP 8 guidelines
- Use Ruff for linting
- Add type hints to all functions

## Testing

- Write tests for all new features
- Maintain test coverage above 80%
- Run the full test suite before submitting PRs

## Questions?

Open an issue or reach out to the maintainers.
